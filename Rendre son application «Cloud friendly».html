Différents niveaux d’abstraction

Les 12 facteurs

  1. Une base de code / plusieurs déploiements
  2. Ne pas dépendre de l’existence implicite de packages au niveau du système
  3. Stocker la configuration dans l’environnement
  4. pas de distinction entre les services locaux et les services tiers
  5. Construction, publication, exécution
  6. Processus
  7. Association de ports
  8. Concurrence
  9. Jetable
  10. Égalité dev/prod
  11. Journalisation

Inconvénients des infrastructures cloud…

  • Impact sur les méthodes de développement
  • Problème de riches
  • Mirroir aux alouettes

Avantages des infrastructures cloud…

  • Élasticité des coûts et des performances
  • Portabilité
  • Maintenance simplifiée
  • …moyennant le respect de certaines règles

Malheureusement… On n’a rien sans rien !

Jetable

<?php
$stdout = fopen('php://stdout', 'w');
$stderr = fopen('php://stderr', 'w');

fwrite($stdout, "Je démarre\n");

$terminate = function () use($stdout) {
  echo fwrite($stdout, "On m’a demandé d’arrêter\n");
  exit(0);
};

pcntl_signal(SIGINT, $terminate);  // Pour la console
pcntl_signal(SIGTERM, $terminate); // Pour docker compose

$i=1;
while($i++) {

  fwrite($stdout, "Début traitement n°{$i}\n");

  for ($j=0;$j<10;$j++) {
      sleep(1);
      if (rand(0,1)) {
          fwrite($stdout, "Action {$j} réussie\n");
      } else {
          fwrite($stdout, "Action {$j} échouée\n");
          fwrite($stderr, "Ça a fait «shhhhhwizzz»\n");
      }
  }
  fwrite($stdout, "Fin traitement n°{$i}\n");
  pcntl_signal_dispatch();
}

      dockerfile: Dockerfile.1

      - MY_HOST

  # Application avec Apache
  app2:
    image: app2
    build:
      dockerfile: Dockerfile.2
    ports:

    environment:
      - MY_HOST

  # Service en ligne de commande

    image: app3

Dépôts séparés

FROM debian:unstable
WORKDIR /srv
RUN apt-get update && \
    apt-get install -yqq git unzip composer php-cli php-curl php-sqlite3 php-dom php-xml curl php-redis php-mysql && \
    curl -fsSL -o /tmp/main.tar.gz https://github.com/symfony/demo/archive/refs/heads/main.tar.gz && \
    tar -xf /tmp/main.tar.gz -C /srv && \
    cd demo-main && \
    composer update --no-progress
CMD ["php", "-d", "variables_order=EGPCS", "-S", "0.0.0.0:8000", "-t", "demo-main/public"]

read_only et tmpfs

version: "3.0"
services:
    demo_prod:
        image: demo
        read_only: true
        ports:
        - "8000:8000"
        environment:
        - APP_ENV=prod
        - APP_DEBUG=false
        - REDIS_HOST=sessions
        - REDIS_PORT=6379
        - DATABASE_URL=mysql://db_user:db_password@db:3306/db_name
        tmpfs:
        - /tmp
        - /srv/demo-main/var/

Développer avec une image Docker

version: "3.0"
services:
  app:
    image: app
    entrypoint: ["php", "cron.php"]
    user: ${UID}:${GID}  # <== évite les soucis de droits
    volumes:
      - .:/srv:ro  # <== protéger son code
    tmpfs:
      - /srv/data # <== pas d’état résiduel
